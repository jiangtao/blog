# Draw.io Diagram Generator

为技术文档和博客文章生成 Draw.io 图表，并保留可持续编辑的 `.drawio` 源文件。

## 使用方法

```text
使用 drawio-diagram 为文章生成技术图表

参数：
- 图表类型 (必需): architecture, workflow, sequence, data-flow, component
- 标题 (必需)
- 描述内容 (必需): 图表的组件、角色、步骤或关系
- 风格 (可选，默认: default)

可用风格:
  default     - 默认自然风格（推荐）
  chalkboard  - 黑板粉笔风格（封面或教学图）
  blueprint   - 技术蓝图
  elegant     - 优雅精致
  minimal     - 极简风格
```

> 默认规则：**先保留本地 Draw.io 源文件，再导出 SVG。**

## 工作流程

1. 判断图表类型（架构图、流程图、时序图、数据流图、组件图）
2. 优先使用 Draw.io 或 Draw.io MCP
3. 创建或保留一份 `.drawio` 源文件作为本地主文件
4. 按清晰网格、明确流向、合理留白完成布局
5. 导出 SVG；只有仓库明确要求时再额外导出 PNG
6. 保存图表文件并在文章中更新引用

## 图表类型

### Architecture

系统技术架构，包含：

- 前端 / 后端组件
- 数据层
- API 或网关
- 外部服务

### Workflow

展示工作步骤和处理流程。

### Sequence

展示角色之间的调用顺序。

### Data Flow

展示数据在不同节点之间的输入、加工和输出。

### Component

展示单个系统或模块内部的结构拆分。

## 风格规范

### Default 默认自然风格

用于文章内技术图表，要求：

- 白色背景
- 柔和配色
- 圆角节点
- 明确箭头
- 保持 `Jerret's Blog` 水印规范（如果仓库已有要求）

### Chalkboard

用于教学氛围或需要和 chalkboard 封面统一的视觉。

### Blueprint

适合系统、架构、流程分析：

- 深蓝背景
- 青色线条
- 网格辅助
- 等宽字体可作为辅助信息

### Elegant / Minimal

分别适用于偏展示型和极简概念型场景。

## 自然线条与间距规则

- 尽量使用 Draw.io 原生路由，而不是手工拼接很多折线
- 连线优先使用圆角或曲线路由，只在必要时使用直角路线
- 保持区块之间有明确分隔，不要节点互相挤压
- 同一层级节点尽量共用对齐线和网格
- 如果文字开始拥挤，应扩大画布或减少节点，不要继续缩字
- 连线周围必须保留足够留白，保证一眼能看出主路径
- 主流程最醒目，次要分支不能压过主流程
- 一个图里颜色不要过多，通常不超过 5-6 个主色

## 输出格式

- **首选**：SVG
- **可编辑主文件**：Draw.io 原生文件
- **可选**：PNG

## 文件保存位置

遵循仓库已有约定；如果仓库没有约定，推荐：

```text
home/public/images/diagrams/
├── <topic>-architecture.svg
├── <topic>-workflow.svg
└── source-local/
    └── <topic>-architecture-v1.0-YYYYMMDD.drawio
```

## 工具规则

- 如果 Draw.io 工具可用，优先使用
- 如果 Draw.io 不可用，可退化成 SVG 或 Mermaid，但结构必须容易回填成 Draw.io
- 不允许为了“好看”引入过多装饰噪音
- 凡是预计后续会调整的图，必须保留 `.drawio`
- 默认不提交 `.drawio`，除非仓库明确要求跟踪该类源文件

## 校验清单

- [ ] 线条自然、间距分明
- [ ] 标签简洁可读
- [ ] `.drawio` 位于本地忽略路径中，除非仓库明确要求跟踪
- [ ] 引用路径与文章内容一致

## 输出

完成时返回：

- SVG 输出路径
- Draw.io 源文件路径
- Draw.io 是否为本地忽略文件
- 图表类型
- 风格
- 是否已更新文章引用
